如果我只记一个图，OpenClaw 的主链路该怎么画？

```
┌─────────┐ ┌─────────────┐ ┌─────────┐ ┌─────────┐ ┌──────────┐ ┌──────────┐
│ Intake │ → │ Context │ → │ Model │ → │ Tools │ → │ Stream │ → │ Persist │
│ (接收) │ │ (上下文组装) │ │ (推理) │ │ (工具执行)│ │ (流式输出) │ │ (持久化) │
└─────────┘ └─────────────┘ └─────────┘ └─────────┘ └──────────┘ └──────────┘
```

**记图口诀**：**收 → 拼 → 想 → 做 → 说 → 存**

- **Intake 收**：用户消息或外部触发进入 [S4]
- **Context Assembly 拼**：把历史记录、工具描述、用户当前输入拼成完整上下文 [S4]
- **Model 想**：LLM 推理决定说什么或调用什么工具 [S4]
- **Tools 做**：如需外部数据（查天气、调 API），执行后结果回注到上下文 [S4]
- **Stream 说**：Token 流式返回给用户，边想边说 [S4]
- **Persist 存**：对话历史、执行结果写入存储，供下次循环使用 [S4]

**关键分叉**：Model 输出若包含工具调用指令，就走 **Tools** 分支执行后再回 Model；若直接是文本，则跳收过 Tools 直接 Stream。
